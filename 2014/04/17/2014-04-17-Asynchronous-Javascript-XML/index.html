Asynchronous Javascript and XML

2014-04-17
College Relics

About ten years ago, Google unveiled the use of a relatively new technology which enabled a client/ server interaction which could track state without a page refresh. The Google application was “Google Maps” and the technology had been dubbed “Ajax” which was an acronym for “Asynchronous Javascript and XML”. “AJAX is the art of exchanging data between client and server, and updating parts of a web page - without reloading the whole page´” (w3schools.com). The main consequence of this is that ajax brings desktop application functionality to the web.

How it works:

• “AJAX is the art of exchanging data between client and server, and updating parts of a web page - without reloading the whole page´” W3 Schools
• It can be viewed as an interface between two (or more) programming languages: Javascript on the client and one of many different languages on the server, ofetn PHP, ASP, Perl, Python or numerous others.
• bringing desktop application functionality to the web
• ASP vs Jquery; library accessibility; MS azure CDN vs Google CDN
• What is normal HTTP processing
• 3 javascript functions are created
• An event occurs which creates a Javascript XMLHttpRequest object. This object provides the coding functionality for the client browser and the web server to exchange data without refreshing the current page. Once created, it sends an “HttpRequest” to the server. Within the request is code which the server responds to by processing data, resending it to the browser (client) and updating the page content.
• Ajax adds a layer to this process. At the beginning of the session, the browser loads an Ajax engine, written in JavaScript and usually concealed in a hidden frame. The engine acts as an intermediary, sending user requests to the server and receiving and rendering the resulting pages. It also processes requests that don’t require server data, such as validation or editing data already in memory. If, for example, a list of items the user is viewing needs sorting, the Ajax engine can do so on the local machine. When server interaction is required, the engine sends the request asynchronously, usually using XML, without interrupting the user. This generates a seamless user experience, relatively unpunctuated by the fits and starts of an ordinary browsing session

II Examples
• Google Maps dragging and dropping; calculating distances

III. Benefits
• Creating web applications similar to desktop applications with user friendly functionality
• migration from form based to GUI based interactivity
• technology is browser independent (provided js is enabled in browser)

IV Variations: JSON vs XML
• If data is to be passed and manipulated by Javascript, the xml process follows as: 1) encoding data into xml form 2) XMLHttpRequest object parse the data into DOM nodes 3 ) Javascrip[t function extracts data
• JSON offers a shorter path whereby the server encodes data into “Javascript object notation” passing it to the browser where it is parsed with an eval() method.
