Chosen Project: Stock Futures Simulation

2014-03-30
RandexTrader

I. Purpose:

This is an educational tool for prospective/existing traders of financial futures contracts. For the purpose of this project, I hope to present a very basic, potentially expansive yet functional website.

Many people consider the stock market to be the largest casino in the world. Critics believe the stock market amounts to “gambling”, and they embrace the “random walk theory”, a belief that, without any inside knowledge of company decisions, that stock price equally likely to rise as fall.

It is NOT the purpose of this site to prove or disprove the random walk theory. Rather, it is an educational tool which generates random trade data and presented in a format that a trader might see on a real trading platform. This is a tool, which can be utilized with no risk of losing money. Players will be able to set the “random” choice of uptick/downtick price generation to “even”, “upward bias” or “downward bias”. Theoretically, if a trader can make consistent (sim) money in a random generated market, s/he should be able to make it in real trading.

III. PRESENTATION:

The Game Page opens with a simple header accross the top and a large expanse of white screen below. In the upper left hand corner, stacked in a vertical table, are the following radio-button controls: “Game speed selection”, ‘fast’, ‘medium’, or ‘slow’. Following these button controls are button controls which include one grouping of “START”; “Pause”; “Resume”; and “Exit”. Below these are another grouping of three different button controls which are different game views.

Once the game commences, a chart is dynamically formed reflecting market price as it moves up or down, one tick at a time. The “pause” and “Resume” controls do not refresh the page. The specific chart is known in the world of trading as a “point and figure” chart. The speed with which the chart is formed is user controlled. After “speed selection” is clicked,”START” is clicked, and graphics, in the form of green rectangles begin to display in the left center portion of the screen.

The three views of the game reflect three different reversal patterns, 15R, 10R, and 5R. For instance, a 15R would be a pattern where the “market” must travel 15 ticks, or rectangle shapes, in one direction before it can show a trend in the opposite direction. The rectangles representing upward ticks are colored green whereas those representing downeward ticks are colored red.

To maintain “state” for these various views, all three charts must be running simultaneously on the server.

User settings include additional elements:
1) Speed selection:  "Fast"; "Medium"; or "Slow".  Any selection here excludes the other two
2) Game controls
3) Market Bias:  "45%"; "50%";  "55%"; Any selection here excludes the other two

The various combination of the above ‘games’ result in nine possible different game levels, for which each will have it’s own “Top Trader” table listings.

The above data elements are derived from an application running on the server.

To give the simulation a taste of current reality in the markets, there will be another Javascript file to access Yahoo for a closing quote for the entity (Russell 2000 index) used in the game. All games on any given date will use this quote as the starting price.

Data Management uses MySQL database:

1) The views of the game are stored in arrays which are callable during any given game
2) Archived games are stored on a per game basis if the profit obtained surpasses a certain threshold for Profit/Loss.  This data, can be called and displayed.
3)  Real time data for the REussell 2000 stock index will be downloaded and displayed in a separate table for comparison purposes. Due to the expense of data, this will only be done for closing and   opening prices initially.

IV. ORGANIZATION/FLOW:

The visitor lands on a page with the same controls as the Main Game Page. However, this page has an explanation of the game and it’s educational importance, mainly to understand markets fully before waging (gambling) money in them. This page has a simple “START” button which begins a truncated version of the game. Should the player want to play a full version of the game (about ten minutes), s/he must enter user information. This information is validated by sending an email with unique password to the user’s email address. Without leaving the Main Page, the user recieves their passowrd and can now enter it into a “password” entry form. Once it is validated, the “Game Page” opens and a game may be played.

Once a game is completed, an alert displays “thank-you for playing”, and after a time delay of 5 seconds, a results page is displayed comparing the game just played with the top 10 or 20 or 50 (not yet determined) results. This will be accompanied with either a “Congratulations, You are in the top x %!” message; OR a “You have not made the cut, but please try again!” message. The results page will contain only two links: 1) “Play Again” 2) Information about trading page.

I believe real traders will gain constructive insight into trading from this website. Beyond the scope of this class project, many developments may be added, such as arbitrage games where two random entities are traded, real historical data, and user controlled programmable rules.
