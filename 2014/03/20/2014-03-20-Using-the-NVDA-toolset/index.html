Grassroots Web Development

Using the NVDA Toolset

2014-03-20
College Relics

There are three main disabilities that command immediate concern of website developers. These are blindness, deafness, motor and cognitive impairment all hindering a users ability to browse the internet.

After downloading the NVDA toolset and visiting three websites,experimenting with the NVDA navigation keys, I was able to make my way around the labeled elements of a page, but only with extreme hesitation and extremely slow. My overall impression was that this is an immensely frustrating experience. Of course, people in general are extremely adaptable to their surroundings and circumstances, and if I had to master this type of web browsing, I would certainly increase my performance; but this adds an entirely new level of complexity to a process which many non-handicapped people find challenging.

For experimentation, I chose these three sites: yahoo.finance, empire state college, and
audiogames.net. I chose Yahoo.finance because it is a content heavy site. At first. the audio streamed so rapidly, I could not understand it. I quickly abandoned the monotonous voice of the generated drone in favor of the microsoft voice. As I took notes, I noticed the reader followed all my keystrokes echoing back each letter as I typed, very confusing. I was able to use the tab key to move from labeled element to labeled element and found the NVDA + down arrow very helpful in reading (hearing) the content. I also noticed that I could interupt the process at any time with my mouse and move it around the screen . If something caught my attention, I could go back to NVDA focus mode. If I missed something, it was easy to shift tab back to where I wanted to be.

The empire college site for SMT course listings wasn’t as busy as Yahoo finance, but still laden with content. Here I noticed I could not use the mouse with the NVDA keys. Also, the site was very top heavy with menu items. If a course selection was clicked, all of these headings had to be tabbed through to get down to the heart of the page.

Audiogames.net a site that is constructed for blind gamers was easily navigable. I could make my way around the site using the tab and shift+tab to go back and forth. With little practice, I was able to choose an audio game and make it to the download page, and then back to the main search game page.

Out of curiosity, I ran each site through the WAVE tool. All had errors and alerts which included noscript elements, redundant title text and redundant links. Yahoo had 27 errors and 74 alerts. Audiogames had only three errors. Empire had 135 errors! Somewhat skeptical about this, I decided to run one of webaim’s own pages through their own evaluation. I chose there own forms practice page, http://webaim.org/articles/nvda/forms.htm. Oddly enough, it had 22 errors, 21 from missing form labels. I was confused though, after examning the source code to find that the labels seemed to all be intact!

One thing I found very interesting was that dynamic page presentation aka Ajax, is often not noticed by the visually impaired and can actually act as a detriment. The screenreader cannot be aware of the updated “dynamic” changes and therefore, new content is likely to be missed. Potential solutions to this problem include ARIA (Accessible Rich Internet Appolications) which make dyunamic content more accessible.

This assignment has made me think of not only making my website accessible, but also of the incredible sensory input/output devices being developed for those people with sensory impairments.
