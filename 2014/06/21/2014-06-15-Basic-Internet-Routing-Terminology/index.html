Basic Internet Routing Terminology

2014-06-21
College Relics

Default Gateway:

The default gateway provides a circuit off the subnet or network often to the internet. This occurs when a router receives a packet with an IP address not in it’s routing table, it then routes the packet to the default gateway.

Packet Forwarding:

The network layer determines the best routes for data (aka packets) flow through the network. The rules by which this is done are defined by the Internet Protocol (IP). IP software is used at each router in the network that handles a particular transmission determining where to send the packets by finding the address of the next router. Integral to this process is the routing table. If the packet contains an address on a router’s table, it will be routed to that device.

A brief description of routing tables:

Routing tables are also called forwarding tables or configuration table; Internet Protocol (IP) routing decisions are based on values contained in the routing tables; in some cases, these values prevent packets from traversing certain circuit portions and in other cases direct packets to travel on certain circuit portions. The size of the table is proportional to the number of networks to which it is connected. The information contained in routing tables allow for 1) balanced loads 2) ability to circumvent malfunctioning equipment in the network

Routing table construction:

Each router has a routing table; each table has at minimum two columns (though it can be configured with more columns and information); in a simple two column table, the first column lists all device IP addresses (including those of other routers) the router can see on the network; the second column lists the interface for each device listed in the first column. The interface is that media onto which the electrical signals (all the packets) are directed by the router.

The routing table is developed by the network manager but in a dynamic routing system it is periodically and continually updated by the individual routers on the network. This allows for the respective routing tables to contain the most efficient routing information though the extra “overhead” of exchanging routing information can impede the speed of the network.

Router communication:

routers are “special-purpose computers” which communicate with a wide variety of devices.[ref]. Aside from the network manager who can connect to it using emulation software for configuration, and Telnet with SSH for maintenance, routers only interface with other devices to move data around. Just like any computer, a router has an operating system which references the config file for directions hundreds of times per second. Once the router is configured, it becomes an autonomous device communicating with other routers using routing protocols.

Routing protocols:

Imagine a routing table with a million IP addresses. It would be all but impossible to efficiently determine the best route through such a system. To ease the burden of routing, networks are “subdivided into autonomous systems of networks” (p 168). Protocols used within an autonomous system are differentiated from those that are used to connect these systems: the former are dubbed “interior routing protocols” while the later are “exterior routing protocols”. The five main Internet Protocols are Border Gateway Protocol (BGP), Internet Control Message Protocol (ICMP), Intermediate System to Intermediate System (IS-IS), Routing Information Protocol (RIP), and Open Shortest Path First (OSPF). RIP is generally used on smaller networks. It is software that tracks the various routes and the number of computers in them. OSPF tracks the state of every network interface in it’s network or subnet.
