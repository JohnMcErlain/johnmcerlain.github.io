Who Will Use My App: Traders or Gamers?

2015-03-11
College Relics

I will eventually need user feedback to determine who the users are. The target audience for the application will be futures traders, but ‘gamers’ may find the app ‘fun’; I will need to differentiate between the traders and the gamers. Moreover, I will be interested in the feedback from both groups as it may provide guidance for the development of the site.

By default,the best data gathering method on-line is the questionnaire. If an app is compelling to users, they have an interest in helping to make it better, and therefore, will spend some time completing the questionnaire. Nonetheless, it should be as brief and succinct as possible to ensure the users are not put off about lengthiness and/or other flaws.

Ultimately, the information gathered is used to aid in the design of the program. The software engineering profession refers to this as ‘requirements elicitation’, which is a fundamental stage in the overall process of software development. Norman’s concept of the “double diamond” is particularly useful here, to point out that defining the problems is more than simply delineating requirements, but itself requires a process of discovery while examining different viewpoints. In my trader simulation, I would like to discover user preferences and overall user experience, whatever they may be.

What type of data would you gather?

To start, the questionnaire would provide me with various quantitative data. Some questions could be simple boolean (true or false). Example: have you ever traded futures contracts on a real platform? Y or N. Others would use a Likert scale, designed on a five point scale. These questions might focus on game controls. Example: Should the ‘execute button’ be 1. on bottom 2. on left 3. no preference 4. on top 5. on right.

Another approach may be to reference the current game layout and ask for opinion, such as to the statement, “The ‘execute button’ is located in the best location”. Do you 1. Strongly agree 2. Somewhat agree 3. No opinion 4. Somewhat disagree 5. Strongly disagree.

and how would you analyze that data?

This would be done using quantitative analysis. I am most interested in ascertaining user satisfaction with the interface but am also quite curious to see if gamers might actually outperform real traders in any way. I would segregate the data between these two groups to first determine if there was any significant differences in trading performance. If there was a divergence, an analysis of survey questions regarding control preferences would ensue using average response bar or pie charts for each group.
